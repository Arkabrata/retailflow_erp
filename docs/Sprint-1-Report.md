## 1. Sprint Overview

**Sprint Duration:** Sprint-1  
**Sprint Goal:**  
To establish the foundation of the RetailFlow system and deliver a working prototype implementing the highest-priority user stories.

Sprint-1 focused on setting up the core backend and frontend architecture, implementing master data modules, and delivering an initial working increment (MVP) that demonstrates the end-to-end procurement foundation.

---

## 2. Sprint-1 Objectives

The objectives defined at the start of Sprint-1 were:

- Establish backend and frontend project structure
- Implement core master data modules
- Build the procurement foundation
- Deliver a functional MVP suitable for sprint review

## 4. Increment Delivered (Working Software)

At the end of Sprint-1, the following working features were demonstrated:

- Item Master with SKU, HSN, pricing details
- HSN Master for tax classification
- Vendor Master for supplier management
- Purchase Order creation workflow
- Goods Receipt Note (GRN) foundation linked to Purchase Orders
- Backend APIs implemented using FastAPI
- Frontend UI implemented using React

This increment represents **Increment-1 (Minimum Viable Product)** of the RetailFlow system.
